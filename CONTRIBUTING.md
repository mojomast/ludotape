# Contributing

Use Node 20+, create a focused branch, and run `npm run check`. Add meaningful `node:test` coverage for behavior changes and update public documentation/formats. Keep runtime and development dependency count at zero unless maintainers explicitly change project scope. Preserve canonical compatibility or document a format-version migration. Do not commit `dist/` or benchmark results.

By contributing, you agree your contribution may be distributed under MIT. Follow the Code of Conduct. Report vulnerabilities through SECURITY.md rather than public issues.
